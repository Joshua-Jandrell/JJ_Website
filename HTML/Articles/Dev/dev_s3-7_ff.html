Finding a framework

17 June 2022
I need to add an external JavaScript Dependency/Framework/Library to my website. But, before I do all that I need to decide what sort of library/framework/dependency do I want? What functions do I need? What will be most useful? It’s time to find out.

Framework vs library

Potaato, potatto?

Libraries are, as their names suggests, a collection of functions [1, 2] You can pick one up off the shelf, read it, and pop it back when you're done. In essence a library is a collection of code snippets with an API of fintions you can use for get a great deal of functionality. Call it from your code and let it do its thing.

If a library is a collection of books, then the framework is the building in which they were housed. (Okay the metaphor is a stretch – but I’ve started with it). You don’t call frameworks – frameworks call you. Structure for your code/website/app/death-ray in accordance with a framework and it will grant you great powers. But, if your code and file structure doesn’t follow it’s rules it won’t understand what you want and will be unable to assist.

So, what do I want?

I have a bt of a challeng ‐

I don't really know.

I’ve already developed quite a bit of JS. Granted it’s not the best, most versatile, most functional, or most efficient. But it’s mine and it does what I want it to do. I get it and it gets me. Do I really want to replace that with a new dependency of the sake of a new dependency?

That’s silly!

Can I make a frame[work] work?

When I comes to frameworks, I have my own charming vanilla homunculus. Look, it’s sure as heck not the Vitruvian man of AngularJS ‐ but it can load files, make templates, and populate my webpage. I’m not making a web-app, I don’t have a backend. Why on earth would I want to get a butler to manage this ramshackle shack of a site?

Besides I think that a framework is great if you are starting 'from scratch'. That way you can built a system to truly capitalise on the framework’s functionality. Less work for you from day one!.

That ship has already sailed. I’ve done the leg work. I don’t see the point of smashing this site into a framework for the sake of an external dependency, I’ll use a fraction of the framework’s functionality and have unnecessary setup work.

But! It has all the functions I want. I know how it works (and what’s wrong with it). This booby trap of a site has everything from asynchronous html injections to the shodow DOM.
