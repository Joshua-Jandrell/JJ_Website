Tedious and fiddly.

Even with my arsenal of custom HTML elements, setting up text like that is tricky. I’ve never written a branching narrative in my life before, let alone one in my own temperamental medium. It’s not easy to plot out and I ended up needing to cut down on quite a lot of it.

Perhaps I should have spent more time writing and less time making widgets like <toggle-block> which I didn’t even end up using because it's still a bit buggy. Actually, Everything is still a bit buggy!

I’m trying to do something new and unusual. Perhaps I am innovating and pushing the envelope. Branching hypertext narratives are interesting.

I’m also doing something that nobody asked for and 99% of the population make do without. What’s wrong with just using twine. I did it all myself but is it worth it?

On a macro scale my site is in the same predicament. It’s got some rather nifty features. You know: automating templet creation with custom HTML elements, it’s own API for importing HTML form other documents, which supports and automated index creation system. I even have my own update-on-scroll function and a code-preview element.

All of these tasks were intreating to tackle and thrilling to solve. But none of them make it a better site.

Yes, hypertext is exciting. But there is a reason that linear narratives still exist: they tell the best stories. I could probably have my point above with fewer words, less buttons and in a shorter amount of time. And it would be clear and concise!

Don’t get me wrong I’m proud of what I’ve done with this site. I pushed myself to the limits of my skills and understanding and one can only ever learn and grow from something like that.

The auto indexing system is less tedious than a static equivalent and I can’t deny the utility of templets as design patterns. I know they're helpful.

I just don’t see direct effects on my site. Infract I was so focused on these system that I dropped the ball with the basics.

I have a poorly edited bracing narrative when a well written paragraph would do?
